# Agglomeration-Platform Rebrand Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rename consort's entire cosmetic identity to the agglomeration-platform vocabulary (`ap` namespace; `hub`/`agent`/`worker`/`cluster`/`DONE`; 10 new command verbs) while preserving the frozen wire protocol byte-for-byte.

**Architecture:** A mechanical-but-judgment-heavy token rename across ~80 shipped files, executed in **3 sequential PRs** (brand → roles → verbs+gate). Each PR keeps `typecheck`+`test`+`lint`+`build`+the stale-token gate green and is independently shippable. `dist/ap.cjs` is **regenerated by `npm run build`**, never hand-edited. The frozen wire protocol (events/sentinel/JSON fields/`contracts.yaml` keys/state filenames/`CLAUDE_CODE_SESSION_ID`) is never touched.

**Tech Stack:** TypeScript (Node/ESM, `.js` import specifiers), esbuild bundle, vitest, eslint, tmux IPC. `tsc --noEmit` is the safety net that catches every broken import after a `git mv`.

**Spec:** `docs/superpowers/specs/2026-06-15-agglomeration-platform-rebrand-design.md`

---

## Global conventions for every task

- **Verification command (the "green bar"):** `npm run typecheck && npm run test && npm run lint && npm run build`. Run it after each rename slice. `typecheck` catches broken imports from file renames; `test` proves behavior unchanged; `build` regenerates `dist/ap.cjs`.
- **Never edit `dist/` by hand.** After `src/` changes, `npm run build` rewrites the bundle; `git add dist/ap.cjs`.
- **File renames use `git mv`** (preserves history), then fix import specifiers (`./old.js` → `./new.js`); `typecheck` lists every miss.
- **The frozen wall — do NOT rename, ever:** event names `ready`/`ack`/`progress`/`done`/`error`/`question`; sentinel `END_OF_INSTRUCTION`; JSON fields `ts`/`summary`/`artifacts`/`note`/`message`/`fatal`/`task_summary`/`model`/`topic`; `contracts.yaml` keys including **`consult_validated`**; state filenames (`status.json`/`pane.json`/`inbox.md`/`outbox`); `CLAUDE_CODE_SESSION_ID`.
- **Generic-English danger tokens** (`part`, `section`, `score`, `perform`, `solo`, `fine`): NEVER blanket-replace. Rename only the identifier/role usages, reviewing each occurrence. Details in the relevant tasks.
- **Stale-token gate is grown incrementally**, one PR's bans per PR, only after that PR's tokens are gone (consistent with the spec's "never ban a still-present token").
